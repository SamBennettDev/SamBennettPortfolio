Spreadsheet
Application

About

This project was a six week long assignment requiring at least 10 hours of work per week. This project was written in C# using windows forms and visual studio IDE. I unfortunately cannot include my source code for the assignment as this assignment will be given to future students. I have included a video showcasing the functionality of the program below.

Week 1

Formula Evaluator

  • Write a method that evaluates arithmetic expressions using standard infix notation. Respect the usual precedence rules. E.G., the string "5+5*2" should evaluate to the integer 15.
  • Package the code into a library.
  • Create unit tests to test the method is working as expected.

Week 2

Dependency Graph

  • Build the infrastructure to handle dependencies between cells by using a dependency graph.
  • Decide which data structures to use to create the dependency graph.
  • Use regression testing and unit tests to test dependency graph is working as expected.

Week 3

Refactor Formula Evaluator

  • Create a more generalized version of the formula evaluator method by creating a Formula class.
  • A Formula object must store the expression to evaluate (e.g., "1+A2") along with any other needed information, and can then be used to "re-evaluate" the expression whenever necessary (e.g., when a dependee changes).
  • Use regression testing and unit tests to test Formula Class is working as expected.

Week 4

Spreadsheet Internals

  • Separate functionality of the program using the MVC architecture. In this case create the "Model" of the spreadsheet application.
  • Abstract the underlying behavior of the spreadsheet (re-evaluating cells based on dependencies) from the user's view of the spreadsheet (the GUI).
  • Use regression testing and unit tests to test spreadsheet is working as expected and achieve 100% code coverage.

Week 5

Completing Spreadsheet Internals

  • Modify, add to, and enhance the implementation of the Spreadsheet class so that it satisfies the specifications in a new given API.
  • Draw useful sketches to represent the system architecture.
  • Use XML to read and write spreadsheet files.
  • Create tests to make sure spreadsheet model is working as expected including reading and writing files.

Week 6

Spreadsheet GUI

  • Create the GUI representing the View/Controller part of the MVC architecture.
  • Use a Task Management System (GitHub Projects) to track assigned work and completion.
  • Create file menu including New, Save, Open, and Close options.
  • Highlight selected cell and include multiple ways to select a different cell (e.g., arrow keys, mouse, etc.).
